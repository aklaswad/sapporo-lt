Sonificator (2011)

Sonificator is a bookmarklet which plays webpages - Liken DOM structure to music score
http://sonificator.com/
(demo)
ATTENTION: Don't look at source code because it's written at the era Web Audio API has only JavaScript Node ;)
I found that Web Page's structure is very simillar to typical music -- both have Intro (<header>), repeating verse/chorus (<section|article|etc...>) and Outro (<footer>) ;)
I call it Data Driven Composition - It could be an Alternative to Algorithmic Composition.
The Fact -- today we can access to tons of random and/or structured data, and they possiblly could be translated into music better than old school algolithmic composition do.
And, now we have enough technologies to do it on user's device without any complicated operation. What they need is only web.

Thoughts about future of web music

Honestly, was bored to see JavaScript versions of Csound, PureData, Max/MSP, SuperColider, etc...
Clones of old school Synthesizers, T*-(303|808|909), ChaosPad, Tenorion, REAKTOR, etc...
Am sick of those clones.
"Hey, listen, My synth is running well on browser, like native code!"
Is this our goal?
I know that Web Music technorogies are making web site and games better.
Is it enough?
What I want is Music.
Music experience which could never been w/o Web Music Technorogies.
I believe Web Music can change the history of music.
Like Dr. Moog, Jim Marshal, and Thomas Edison did at 20th century.
Like Recording, Scoring, and Equal Temperament.
Let's think not only about code
but also the future of Music
